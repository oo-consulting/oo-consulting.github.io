Overview

고도화 개요

기본적인 문서 검색 및 답변 생성 기능을 넘어, 정확도와 확장성, 그리고 사용자 경험을 극대화하는 것을 목표로 합니다.

Phase 1 (1~2개월)

검색 품질 강화

하이브리드 검색 도입과 쿼리 확장을 통해 정확한 매칭이 필요한 질의의 성능을 대폭 향상시킵니다.

🔍

하이브리드 검색

벡터 유사도 검색(의미) + BM25 키워드 검색(매칭)을 결합하여 고유명사 검색 성능 개선

🔄

쿼리 확장 (Query Expansion)

LLM을 활용해 사용자 질문을 재작성하고 유사/연관 질문으로 확장하여 검색 범위 확대

✂️

청킹 전략 고도화

단순 고정 크기 분할에서 의미 단위(Semantic) 및 계층적 청킹으로 문맥 보존 강화

class HybridRetriever: def search(self, query: str): # 1. 의미적 유사도 검색 (Dense) vector_results = self.vector_search(query) # 2. 키워드 매칭 검색 (Sparse) keyword_results = self.bm25_search(query) # 3. 상호 순위 융합 (RRF) return self.reciprocal_rank_fusion( vector_results, keyword_results )
🚀 기대 효과
  • • 고유명사, 코드 변수명 검색 정확도 향상
  • • 단순 키워드 매칭 실패 감소
  • • 문맥 연결성 강화
Phase 2 (2~3개월)

LLM 성능 최적화

더 강력한 모델과 정교한 프롬프트 엔지니어링으로 답변 품질과 응답 속도를 개선합니다.

용도 현재 모델 권장 업그레이드
일반 질의 qwen2.5:3b qwen2.5:7b / llama3.1:8b
코드 분석 coder:6.7b deepseek-coder:16b
한국어 특화 - EEVE-Korean-10.8B
⚡

스트리밍 응답 (SSE)

토큰 단위 실시간 출력으로 체감 응답 속도 50% 이상 향상

# 개선된 시스템 프롬프트 구조 1. **역할 정의 (Role Definition)** 전문가 페르소나 부여 2. **컨텍스트 활용 (Context Usage)** 제공된 문서(Context) 내에서만 답변 엄수 3. **응답 형식 (Output Format)** 가독성 높은 구조화된 마크다운 포맷 4. **인용 규칙 (Citation Rules)** [문서명](페이지) 형태의 출처 명시 필수 5. **불확실성 표현 (Uncertainty)** 정보가 부족할 경우 솔직하게 "알 수 없음" 표기
Phase 3 (2~3개월)

데이터 파이프라인 강화

실시간 데이터 동기화와 고품질 전처리로 지식의 최신성과 품질을 보장합니다.

🔄

실시간 인덱싱

파일 시스템 감지(FileWatcher)를 통해 문서 변경 즉시 인덱스 업데이트
[File] → [Queue] → [Indexer] → [DB]

🏷️

메타데이터 확장

작성자, 부서, 생성일, 태그, 보안 등급 등 풍부한 메타데이터 관리로 필터링 검색 지원

📑

고급 전처리

OCR(이미지 텍스트), 표 구조화, 이미지 캡션 생성으로 비정형 데이터 처리 능력 강화

Phase 4 (1~2개월)

사용자 경험(UX) 개선

사용자가 더 쉽고 직관적으로 정보를 탐색할 수 있는 인터페이스를 제공합니다.

대화형 UI 고도화

멀티턴 대화(문맥 유지), 후속 질문 자동 제안, 답변 피드백(좋아요/싫어요)

고급 검색 필터

소스 유형, 날짜 범위, 부서별, 태그별 상세 검색 필터 제공

시각화 대시보드

지식베이스 통계, 자주 묻는 질문(FAQ) 트렌드, 검색어 분석

🖼️

Advanced Dashboard

Knowledge Graph & Analytics View

Phase 5 & 6 (Long Term)

엔터프라이즈 및 고급 기술

보안, 관리 기능 강화 및 최신 AI 에이전트 기술 도입

👮

접근 제어 (RBAC)

사용자 역할(Admin, Editor, Viewer)에 따른 문서 접근 권한 및 기능 제어

📜

감사 로깅

누가, 언제, 무엇을 검색했고 어떤 문서를 열람했는지에 대한 전체 감사 로그 기록

🤖

Agentic RAG

질문 계획(Planner) → 단계별 검색 → 자체 검증(Reflection)을 수행하는 자율 에이전트

🕸️

GraphRAG

문서 간 관계를 지식 그래프로 구축하여 복잡한 연결 고리를 추론하고 설명
