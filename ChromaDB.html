ChromaDB 벡터 데이터베이스 구축

KorBERT 임베딩과 메타데이터를 ChromaDB 컬렉션에 적재하고, HNSW 기반 인덱스를 튜닝하여 RAG 검색 품질을 극대화합니다.

개발 순서

3단계 · ChromaDB

컬렉션 스키마 · HNSW 파라미터 · 검색 검증

단계 개요

ChromaDB는 경량이면서 확장성이 뛰어난 벡터 데이터베이스입니다. 컬렉션 구조를 명확히 정의하고 HNSW 인덱스를 환경에 맞게 조정하면 검색 지연을 줄이면서도 높은 정확도를 유지할 수 있습니다.

배치 적재와 스트리밍 적재를 병행하고, 쿼리 성능·Hit ratio·메모리 사용량을 지속적으로 관찰하는 것이 핵심입니다.

구현 상세

구분 내용 비고
컬렉션 구조 namespace · version · language 분리 메타데이터: 파일 경로, 심볼, 태그
인덱스 설정 HNSW (cosine) / M=32 / ef=200 쿼리 패턴에 따라 ef_search 조정
적재 파이프라인 배치(야간) + 스트리밍(변경 이벤트) Airflow · Kafka 연동
모니터링 Prometheus · Grafana · Alertmanager 지연 시간·Hit ratio·메모리 사용량

설치 및 개발 방법

1. 서버 실행

pip install chromadb[server] uvicorn fastapi
chromadb server --path ./chroma_store

# Docker
ocker run -d --name chroma -p 8000:8000 ghcr.io/chroma-core/chroma:latest

2. 임베딩 적재

python pipelines/chromadb_load.py \
  --collection korbert-code-v1 \
  --embedding artifacts/korbert_embeddings.jsonl \
  --metadata artifacts/metadata.json \
  --distance cosine

3. 검색 검증

python evaluations/search_sanity.py \
  --collection korbert-code-v1 \
  --queries datasets/sample_queries.json \
  --top-k 5
