ChromaDB

벡터 데이터베이스 구축과 검색 인덱스 최적화

개발 순서 3단계

개요

ChromaDB 단계는 KorBERT에서 생성된 임베딩을 고성능 벡터 데이터베이스에 적재하여 검색을 위한 인덱스를 구성하는 과정입니다. 컬렉션 스키마 설계, 파티셔닝 전략, 메타데이터 필터링, 인덱스 빌드 자동화가 핵심 업무입니다.

운영 환경에서는 벡터 데이터의 증분 적재가 자주 발생하므로, 배치 파이프라인과 실시간 파이프라인을 분리 설계하고, 거리 함수(코사인, 점곱 등)와 임계값을 조정해 검색 품질을 튜닝합니다.

컬렉션 스키마 정의 벡터 인덱스 구축 메타데이터 필터링 증분 적재 자동화

구현 상세

구분 내용 상세 설명
컬렉션 설계 Namespace/Collection 구조 KorBERT 벡터를 서비스 도메인, 버전, 언어별로 분리해 컬렉션을 구성합니다. 메타 필드(파일 경로, 심볼, 태그, 임베딩 버전)를 스키마에 포함합니다.
적재 전략 배치/스트리밍 파이프라인 병행 대량 배치는 nightly job으로, 변경 파일은 이벤트 기반 스트리밍으로 처리하여 최신 상태를 유지합니다.
인덱스 관리 HNSW 파라미터 튜닝 M, ef_construction, ef_search 값을 데이터 규모에 맞게 조정해 검색 정확도와 속도의 균형을 맞춥니다.
운영 모니터링 Latency & Recall 추적 Prometheus/Grafana 연동으로 쿼리 지연 시간과 재현율을 모니터링하고 임계값을 초과하면 알림을 발생시킵니다.

설치 및 개발 방법

1. ChromaDB 환경 설치

# 가상환경 준비
python -m venv .venv
source .venv/bin/activate  # Windows: .venv\Scripts\activate

# ChromaDB 및 필수 의존성 설치
pip install chromadb[server] uvicorn fastapi
pip install "chromadb>=0.5.0" --upgrade

서버 모드로 운영할 계획이라면 chromadb[server] 옵션을 사용해 REST API 구성 요소를 함께 설치합니다.

2. 컬렉션 생성 및 적재 스크립트

python scripts/create_collection.py \
  --collection kor-project-v1 \
  --dimension 768 \
  --distance cosine \
  --metadata-schema ./schemas/korbert_metadata.json

python scripts/load_embeddings.py \
  --collection kor-project-v1 \
  --embeddings ./artifacts/embeddings \
  --metadata ./artifacts/metadata.json \
  --batch-size 500

KorBERT 임베딩의 차원 수(예: 768)에 맞춰 컬렉션을 생성하고, 메타데이터 스키마 파일을 통해 필수 필드를 검증합니다.

3. 인덱스 최적화 및 테스트

python scripts/tune_index.py \
  --collection kor-project-v1 \
  --index-type hnsw \
  --m 32 \
  --ef-construction 200 \
  --ef-search 64

python scripts/query_samples.py \
  --collection kor-project-v1 \
  --query "KorBERT 토큰화 전략" \
  --top-k 5

인덱스 파라미터 튜닝 후, 대표 쿼리 샘플을 통해 평균 반응 시간과 검색 품질을 검증합니다. 필수 KPI: P@5 ≥ 0.8, 평균 지연 < 200ms.

운영 아키텍처 체크포인트

배포 토폴로지

  • ChromaDB 서버 다중 인스턴스 구성 (Active-Active)
  • Load Balancer를 통한 쿼리 분산과 Health Check 구성
  • Persistent Volume을 사용해 노드 교체 시 데이터 유지

보안 및 거버넌스

  • API Key 기반 인증과 역할별 접근 제어(Role-based ACL)
  • 민감 데이터 필드를 별도 암호화 저장 (예: 고객 식별자)
  • Audit Log를 S3/Blob으로 주기적 백업

운영 자동화

  • 증분 임베딩 배치를 Airflow DAG으로 관리
  • 메타데이터 검증 실패 시 Slack/Teams 알림 발송
  • 주기적 백업 및 복구 시나리오 리허설

성능 튜닝

  • 쿼리 캐시와 샤딩을 활용한 대량 질의 대응
  • ef_search를 트래픽 패턴에 맞춰 동적으로 조정
  • 백오프 재시도 로직으로 장애 전파 최소화

RAG 시스템에서의 장점

정확한 문맥 검색

코드와 설명을 함께 보관해 함수 목적과 사용 예제를 동시에 반환합니다.

고성능 확장성

HNSW 기반 인덱스와 샤딩 구조로 대규모 임베딩을 안정적으로 처리합니다.

탄력적 운영

배치/실시간 파이프라인을 분리해 변경 사항을 빠르게 반영하면서 가용성을 유지합니다.

거버넌스 대응

메타데이터 기반 필터로 민감 정보 접근을 제한하고 감사 로그를 보존할 수 있습니다.
