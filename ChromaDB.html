ChromaDB 벡터 데이터베이스 구축

KorBERT에서 생성한 임베딩을 ChromaDB에 적재하고, 검색 인덱스를 구성하여 RAG 질의에 즉시 사용할 수 있는 환경을 구축합니다.

현재 진행률

70% · 인덱스 튜닝 중

HNSW 파라미터, 메타데이터 필터링 테스트 진행

개요

ChromaDB는 경량이면서 확장성이 우수한 벡터 데이터베이스로, RAG 시스템에서 임베딩 검색을 담당합니다. 컬렉션 구조와 메타데이터 스키마를 명확히 정의하면, LLM 응답에 필요한 컨텍스트를 신속하고 정확하게 검색해올 수 있습니다.

본 단계는 임베딩 등록·검색·검증까지 이어지는 자동화 파이프라인을 마련하고, HNSW 기반 인덱스를 환경에 맞게 튜닝하는 것이 핵심입니다.

구현 상세

항목 내용 비고
컬렉션 스키마 namespace · version · language 로 파티션 메타데이터: 파일 경로, 심볼, 태그
인덱스 타입 HNSW (cosine 거리) M=32, ef_construction=200 권장
적재 파이프라인 배치(야간) + 스트리밍(실시간 변경) Airflow / Kafka 연동 가능
모니터링 Prometheus · Grafana · Alertmanager 쿼리 지연, Hit ratio, 메모리 사용량 추적

설치 및 개발 방법

1. ChromaDB 서버 모드 실행

# 로컬 테스트
pip install chromadb[server] uvicorn fastapi
chromadb server --path ./chroma_store

# Docker 예시
docker run -d --name chroma -p 8000:8000 ghcr.io/chroma-core/chroma:latest

2. 임베딩 적재 스크립트

python pipelines/chromadb_load.py \
  --collection korbert-code-v1 \
  --embedding artifacts/korbert_embeddings.jsonl \
  --metadata artifacts/metadata.json \
  --distance cosine

3. 검색 품질 테스트

python evaluations/search_sanity.py \
  --collection korbert-code-v1 \
  --queries datasets/sample_queries.json \
  --top-k 5

운영 체크리스트

고가용성

  • Master/Replica 구성 또는 Sharding 고려
  • Persistent Volume으로 인덱스 영속화
  • 배포 시 Rolling Update + Health Check

보안

  • API Key 기반 접근 제어 및 IP 화이트리스트
  • 민감 메타데이터는 암호화 후 저장
  • 감사 로그(Audit Log) 정기 백업

성능 최적화

  • 쿼리 패턴별 ef_search 값 튜닝
  • 대량 upsert 전 배치 분할 및 압축
  • 샤딩 시 동일 namespace 균등 분배

품질 관리

  • Top-K 검색 결과와 레이블 데이터 비교
  • 정기 리프레시 작업을 통해 오래된 임베딩 폐기
  • 백업 스냅샷 생성 및 복구 시나리오 테스트
