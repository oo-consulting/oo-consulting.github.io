LlamaIndex Orchestrator 구축

Parser → KorBERT → ChromaDB → LLM으로 이어지는 파이프라인을 LlamaIndex QueryEngine으로 통합하고, 평가·관측·피드백 루프를 갖춘 RAG 워크플로 엔진을 구성합니다.

개발 순서

5단계 · Orchestrator

Query Pipeline · Router · Evaluator · Observability

단계 개요

LlamaIndex Orchestrator는 Retriever, Synthesizer, LLM, Evaluator를 모듈로 구성하여 일관된 질의 처리 흐름을 제공합니다. QueryEngine에 Router를 적용해 도메인별로 다른 파이프라인을 사용하고, CallbackManager와 Observability를 통해 운영 지표를 수집합니다.

정기적으로 평가를 수행하고 피드백을 기록하면, 품질 슬립을 빠르게 감지하고 개선할 수 있습니다.

구현 상세

항목 내용 비고
Query Pipeline ChromaDBRetriever → ReActSynthesizer 검색·응답 통합
Router RouterQueryEngine으로 도메인 분기 코드/FAQ/문서별 프롬프트
평가 자동화 ResponseEvaluator + 규칙 기반 검증 정확도·출처·포맷
Observability CallbackManager + OpenTelemetry 지연, 실패율, 토큰 사용량

설치 및 개발 방법

1. 의존성 설치

python -m venv .venv
source .venv/bin/activate
pip install llama-index-core llama-index-llms-ollama
pip install llama-index-vector-stores-chroma llama-index-embeddings-openai pyyaml

2. 파이프라인 구성

python pipelines/build_query_engine.py \
  --config configs/pipeline.yaml \
  --output artifacts/query_engine.json

3. 평가 실행

python evaluations/run_rag_eval.py \
  --engine artifacts/query_engine.json \
  --dataset datasets/rag_benchmark.json \
  --metrics exact_match,faithfulness,latency
