개요

LlamaIndex Orchestrator는 Retriever, Synthesizer, Evaluator, Callback을 조합하여 일관된 질의 처리 흐름을 제공합니다. 각각의 단계를 모듈화하면 구성 변경과 확장에 유연하게 대응할 수 있습니다.

본 단계에서는 QueryEngine 설계, Router를 활용한 도메인 분기, LLM 기반 평가 자동화, 관측(Observability) 체계를 함께 구축합니다.

구현 상세

항목 내용 비고
Query Pipeline Retriever → Synthesizer → LLM 응답 ChromaDBRetriever + ReActSynthesizer
Router RouterQueryEngine으로 도메인별 엔진 선택 코드/FAQ/문서별로 다른 프롬프트 적용
평가 자동화 ResponseEvaluator + 규칙 기반 검증 정확도·출처·포맷 평가 점수 수집
Observability CallbackManager + OpenTelemetry 지연/오류/토큰 사용량 실시간 추적

설치 및 개발 방법

1. 패키지 설치

python -m venv .venv
source .venv/bin/activate  # Windows: .venv\Scripts\activate

pip install llama-index-core llama-index-llms-ollama
pip install llama-index-vector-stores-chroma llama-index-embeddings-openai
pip install pyyaml

2. 파이프라인 구성

python pipelines/build_query_engine.py \
  --config configs/pipeline.yaml \
  --output artifacts/query_engine.json

3. 평가 실행

python evaluations/run_rag_eval.py \
  --engine artifacts/query_engine.json \
  --dataset datasets/rag_benchmark.json \
  --metrics exact_match,faithfulness,latency

운영 체크리스트

버전 관리

  • 파이프라인 설정을 Git/DVC로 버전 관리
  • Retriever/LLM 조합 변경 시 릴리스 노트 작성
  • Semantic Versioning으로 영향 범위 명시

배포 전략

  • Blue/Green 혹은 Canary 배포로 리스크 최소화
  • 배포 전 Smoke 테스트 자동 실행
  • 토큰 사용량 예산을 사전 정의하고 모니터링

피드백 루프

  • 사용자 평점, 수정 제안을 메타데이터와 함께 저장
  • 부정 피드백은 주간 회의에서 원인 분석
  • 재학습용 데이터셋을 자동으로 수집

장애 대응

  • 외부 의존성 장애 시 캐시된 응답 사용
  • Trace ID를 기반으로 문제 재현 및 분석
  • 장애 사례를 테스트 케이스에 즉시 반영
