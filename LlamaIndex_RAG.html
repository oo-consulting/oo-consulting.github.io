LlamaIndex Orchestrator 구축

Parser → KorBERT → ChromaDB → LLM으로 이어지는 파이프라인을 LlamaIndex QueryEngine으로 통합하고, 평가/관측/피드백 루프를 통해 안정적인 RAG 워크플로를 구현합니다.

단계 개요

LlamaIndex Orchestrator는 Retriever, Synthesizer, LLM, Evaluator를 모듈 단위로 조합하여 질의 처리 흐름을 표준화합니다. RouterQueryEngine을 사용하면 도메인별로 서로 다른 파이프라인을 선택할 수 있습니다.

ResponseEvaluator와 CallbackManager를 연동해 품질 지표와 운영 지표를 수집하고, 피드백 루프를 통해 지속적으로 개선합니다.

구현 상세

항목 내용 비고
Query Pipeline ChromaDBRetriever → ReActSynthesizer 검색 결과와 LLM 응답 통합
Router RouterQueryEngine으로 도메인 분기 코드/FAQ/문서별 프롬프트
평가 자동화 ResponseEvaluator + 규칙 기반 검증 정확도·출처·형식 점검
Observability CallbackManager + OpenTelemetry 지연·실패율·토큰 사용량 추적

설치 및 개발 방법

1. 의존성 설치

python -m venv .venv
source .venv/bin/activate
pip install llama-index-core llama-index-llms-ollama
pip install llama-index-vector-stores-chroma llama-index-embeddings-openai pyyaml

2. 파이프라인 구성

python pipelines/build_query_engine.py \
  --config configs/pipeline.yaml \
  --output artifacts/query_engine.json

3. 평가 실행

python evaluations/run_rag_eval.py \
  --engine artifacts/query_engine.json \
  --dataset datasets/rag_benchmark.json \
  --metrics exact_match,faithfulness,latency
