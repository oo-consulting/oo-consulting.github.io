Redis 기반 RAG 캐시·세션 관리

Redis로 응답 캐시, 사용자 세션, 레이트 리미트를 관리하여 LLM 호출 비용을 절감하고 시스템 안정성을 높입니다.

개발 순서

8단계 · Redis

응답 캐시 · 세션 · 레이트 리미트

단계 개요

Redis는 RAG 시스템의 캐시, 세션, 레이트 리미트 저장소 역할을 담당합니다. 자주 반복되는 질문, 대화 세션, API 사용량을 저장해 성능과 비용을 동시에 개선합니다.

Sentinel/Cluster 구성으로 고가용성을 확보하고, 백업/복구, 보안, 모니터링 정책을 세워 운영 안정성을 높입니다.

구현 상세

항목 내용 비고
응답 캐시 질의+컨텍스트 해시를 키로 사용 TTL 5~15분, LLM 호출 감소
세션 저장소 Hash 구조로 대화 히스토리 유지 Web App 즐겨찾기 연동
레이트 리미트 Lua 기반 토큰 버킷 429 응답, 고객별 정책
고가용성 Sentinel / Cluster Persistent Volume으로 백업

설치 및 개발 방법

1. Redis 설치

docker run -d --name redis-rag -p 6379:6379 redis:7-alpine
# Helm (고가용성 예시)
helm repo add bitnami https://charts.bitnami.com/bitnami
helm install redis bitnami/redis --set replica.replicaCount=2 --set auth.enabled=true

2. 파이썬 연동

from redis.asyncio import Redis
redis = Redis(host=..., port=..., password=...)

async def get_cache(key: str):
    data = await redis.get(key)
    return json.loads(data) if data else None

async def set_cache(key: str, value: dict, ttl: int = 600):
    await redis.set(key, json.dumps(value), ex=ttl)

3. 레이트 리미트 Lua

RATE_LIMIT_SCRIPT = """
local current = redis.call('incr', KEYS[1])
if tonumber(current) == 1 then
  redis.call('expire', KEYS[1], ARGV[1])
end
if tonumber(current) > tonumber(ARGV[2]) then
  return 0
end
return current
"""
