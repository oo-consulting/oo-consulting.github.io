Redis Cache

RAG 응답 캐싱과 세션/레이트 리미트 운영 전략

개발 순서 8단계

개요

Redis는 RAG 시스템에서 응답 캐시, 세션 저장소, 레이트 리미트 저장소 역할을 담당합니다. FastAPI 게이트웨이와 Web App에서 발생하는 반복 요청을 캐시하여 LLM/ChromaDB 호출 비용을 절감하고, 사용자 세션과 속도 제한을 중앙에서 관리합니다.

고가용성을 위해 Redis Cluster 또는 Redis Sentinel을 구성하고, TTL/만료 정책과 백업 전략을 통해 데이터 일관성을 유지합니다.

응답 캐싱 세션 저장소 레이트 리미트 고가용성

구현 상세

구분 내용 상세 설명
응답 캐시 LLM 결과 저장 질문 + 검색 컨텍스트 해시를 키로 활용해 5~15분 TTL로 캐시합니다. 동일한 질문 재요청 시 Redis에서 즉시 반환합니다.
세션 관리 웹/모바일 세션 FastAPI/Web App의 세션 정보를 Redis Hash에 저장하고, 사용자별 대화 히스토리를 보조 저장소로 관리합니다.
레이트 리미트 Token Bucket 사용자 또는 API Key별로 초당/분당 호출 횟수를 체크하고, 초과 시 429 응답을 반환합니다. Lua 스크립트로 원자성을 보장합니다.
고가용성 Sentinel / Cluster Sentinel로 마스터 장애를 자동 감지하여 페일오버하고, 트래픽이 많은 경우 3노드 이상으로 Cluster를 구성합니다.

설치 및 개발 방법

1. Redis 설치

# 로컬 테스트 (Docker)
docker run -d --name redis-rag -p 6379:6379 redis:7-alpine

# 프로덕션 예시 (Helm chart)
helm repo add bitnami https://charts.bitnami.com/bitnami
helm install redis bitnami/redis --set replica.replicaCount=2 --set auth.enabled=true

클라우드 서비스(AWS Elasticache, Azure Cache for Redis 등)를 사용할 경우 VPC 구성과 보안 그룹을 설정합니다.

2. FastAPI 연동 예시

from redis.asyncio import Redis

redis = Redis(host=settings.REDIS_HOST, port=settings.REDIS_PORT, password=settings.REDIS_PASSWORD)

async def get_cached_response(key: str):
    raw = await redis.get(key)
    return json.loads(raw) if raw else None

async def set_cached_response(key: str, value: dict, ttl: int = 900):
    await redis.set(key, json.dumps(value), ex=ttl)

Lua 스크립트를 사용해 캐시된 응답과 메타데이터를 동시에 갱신할 수 있습니다.

3. 레이트 리미트 Lua 스크립트

RATE_LIMIT_SCRIPT = """
local current = redis.call('incr', KEYS[1])
if tonumber(current) == 1 then
  redis.call('expire', KEYS[1], ARGV[1])
end
if tonumber(current) > tonumber(ARGV[2]) then
  return 0
end
return current
"""
token = await redis.eval(RATE_LIMIT_SCRIPT, keys=[bucket_key], args=[window, limit])
if token == 0:
    raise HTTPException(status_code=429, detail='Rate limit exceeded')

API Key별/사용자 ID별 버킷을 분리해 차별화된 속도 제한 정책을 운영합니다.

운영 및 거버넌스 체크리스트

모니터링

  • Keyspace hits/misses, 메모리 사용량, evicted keys 추적
  • Redis Exporter + Prometheus + Grafana 대시보드 구성
  • 메모리 임계값(예: 80%) 초과 시 슬랙 알림

백업 및 복구

  • RDB 스냅샷과 AOF(Append Only File)를 병행
  • 주기적으로 백업 파일을 Object Storage(S3 등)에 저장
  • 복구 시나리오를 분기별로 리허설

보안

  • AUTH 비밀번호 설정 및 TLS 암호화 적용
  • iptable/VPC로 허용된 서브넷만 접근 허용
  • 관리 명령(CONFIG, FLUSH 등)은 ACL로 제한

데이터 거버넌스

  • 캐시 TTL 정책 문서화 및 서비스별 표준 가이드 정의
  • PII/민감 데이터는 캐시하지 않도록 필터링
  • 주기적으로 오래된 키/사용되지 않는 키를 정리

RAG 시스템에서의 장점

응답 속도 향상

자주 조회되는 질문/컨텍스트를 캐시에 저장해 LLM 호출을 줄이고 지연 시간을 단축합니다.

비용 절감

LLM·벡터 검색 호출 수를 줄여 토큰 비용과 클라우드 인프라 비용을 절감합니다.

운영 안정성

세션/레이트 리미트 관리로 폭주 트래픽을 제어하고 장애 전파를 방지합니다.

확장 용이성

클러스터링과 샤딩을 통해 트래픽 증가에도 안정적으로 확장할 수 있습니다.
