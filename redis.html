Redis 기반 캐시·세션·레이트 관리

RAG 응답 캐시, 사용자 세션, 레이트 리미트를 Redis로 통합 관리하여 성능과 안정성을 동시에 확보합니다.

현재 진행률

45% · 캐시 전략 수립 중

응답 캐시 TTL · 토큰 버킷 레이트 리미트 설계

개요

Redis는 RAG 시스템에서 반복되는 질의를 캐시하고, 사용자 세션과 레이트 리미트를 추적하는 핵심 인프라입니다. KorBERT/ChromaDB/LLM 호출 비용을 줄이고 장애 시에도 빠르게 응답을 제공할 수 있도록 돕습니다.

고가용성을 위해 Sentinel 또는 Cluster 구성을 옵션으로 두고, 백업·모니터링·보안을 함께 설계해야 합니다.

구현 상세

항목 내용 비고
응답 캐시 질의+컨텍스트 해시로 저장 TTL 5~15분, LLM 호출 감소
세션 저장소 Hash 구조로 사용자 상태 유지 Web App 즐겨찾기/히스토리 복원
레이트 리미트 Lua 기반 토큰 버킷 429 응답, 고객별 정책 지원
고가용성 Sentinel / Cluster 구성 Persistent Volume으로 데이터 유지

설치 및 개발 방법

1. Redis 설치

# 로컬 테스트
docker run -d --name redis-rag -p 6379:6379 redis:7-alpine

# Helm (고가용성 예시)
helm repo add bitnami https://charts.bitnami.com/bitnami
helm install redis bitnami/redis --set replica.replicaCount=2 --set auth.enabled=true

2. 파이썬 연동

from redis.asyncio import Redis

redis = Redis(host=..., port=..., password=...)

async def get_cache(key: str):
    data = await redis.get(key)
    return json.loads(data) if data else None

async def set_cache(key: str, value: dict, ttl: int = 600):
    await redis.set(key, json.dumps(value), ex=ttl)

3. 레이트 리미트 Lua

RATE_LIMIT_SCRIPT = """
local current = redis.call('incr', KEYS[1])
if tonumber(current) == 1 then
  redis.call('expire', KEYS[1], ARGV[1])
end
if tonumber(current) > tonumber(ARGV[2]) then
  return 0
end
return current
"""
