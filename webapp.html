Web App 사용자 인터페이스 구축

FastAPI 게이트웨이와 연동해 실시간 스트리밍 응답, 근거 문서, 사용자 피드백을 제공하는 RAG 웹 애플리케이션을 구현합니다.

개발 순서

7단계 · Web App

Next.js · 스트리밍 UI · 접근성 · 다국어

단계 개요

Web App은 RAG 시스템을 사용자에게 제공하는 프론트엔드입니다. Next.js + Tailwind를 사용해 빠르게 개발하고, React Query와 Zustand로 데이터·UI 상태를 분리 관리합니다.

스트리밍 응답, 근거 문서, 피드백 수집, 접근성, 다국어 지원을 고려해 UX를 설계합니다.

구현 상세

항목 내용 비고
프레임워크 Next.js + Tailwind CSS CSR + SSR 혼합 구성
상태 관리 React Query + Zustand API 캐시 vs UI 상태 분리
스트리밍 EventSource / WebSocket 토큰 단위 렌더링, 타이핑 효과
접근성 ARIA 라벨, 키보드 네비게이션 다국어 & 다크모드

설치 및 개발 방법

1. 프로젝트 초기화

npx create-next-app@latest rag-webapp --typescript --tailwind --eslint
cd rag-webapp
npm install @tanstack/react-query zustand react-markdown rehype-highlight highlight.js

2. API 연동 훅

export const useRagQuery = () => {
  const client = useQueryClient();
  return useMutation({
    mutationFn: async (payload: RagRequest) => {
      const res = await fetch('/api/rag/query', {
        method: 'POST',
        headers: { 'Content-Type': 'application/json' },
        body: JSON.stringify(payload),
      });
      if (!res.ok) throw new Error('Query failed');
      return res.json();
    },
    onSuccess: data => client.setQueryData(['history'], prev => [...(prev ?? []), data]),
  });
};

3. 배포 파이프라인

npm run build
vercel deploy --prod

# 사내망 Ddocker 배포
docker build -t registry.example.com/rag-webapp:latest .
docker push registry.example.com/rag-webapp:latest
kubectl rollout restart deployment rag-webapp
