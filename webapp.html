Web Application

RAG 사용자 인터페이스 설계와 프론트엔드 운영 전략

개발 순서 7단계

개요

Web App 단계는 FastAPI 게이트웨이와 연동해 사용자 친화적인 RAG 인터페이스를 제공합니다. 채팅형 UX, 스트리밍 응답, 코드 하이라이팅, 근거 문서 표시 등 현업/개발자 요구를 모두 반영하여 생산성을 극대화합니다.

React 기반 SPA 또는 Next.js를 활용해 CSR과 SSR을 조합하고, 인증 상태 동기화, 캐싱 전략, 로딩 경험을 고려한 컴포넌트 설계를 진행합니다.

대화형 UX 스트리밍 표시 근거 문서 노출 반응형/접근성

구현 상세

구분 내용 상세 설명
프레임워크 Next.js + Tailwind SEO가 필요한 페이지는 SSR, 대화형 화면은 CSR로 구성하고 Tailwind로 디자인 시스템을 공유합니다.
상태 관리 React Query + Zustand API 응답 및 캐시는 React Query로, UI 상태(모달, 테마)는 Zustand로 관리해 로직을 분리합니다.
스트리밍 EventSource/WebSocket FastAPI WebSocket과 연결해 토큰 단위 응답을 실시간 표시하고, 타이핑 효과·진행률을 제공해 몰입감을 높입니다.
접근성 ARIA + 키보드 네비게이션 버튼/입력 필드에 ARIA 라벨을 추가하고 키보드만으로도 주요 기능을 사용할 수 있도록 포커스 순서를 정의합니다.

설치 및 개발 방법

1. 프로젝트 초기화

npx create-next-app@latest rag-webapp --typescript --eslint --tailwind
cd rag-webapp
npm install @tanstack/react-query zustand react-markdown rehype-highlight highlight.js

디자인 시스템 문서화를 위해 Storybook을 추가하면 컴포넌트 재사용성을 높일 수 있습니다.

2. API 연동 훅 작성

// src/hooks/useRagQuery.ts
export const useRagQuery = () => {
  return useMutation({
    mutationFn: async (payload: RagRequest) => {
      const res = await fetch('/api/rag/query', {
        method: 'POST',
        headers: { 'Content-Type': 'application/json' },
        body: JSON.stringify(payload),
      });
      if (!res.ok) throw new Error('Query failed');
      return res.json();
    },
  });
};

스트리밍 응답이 필요하면 EventSource 또는 socket.io-client로 WebSocket을 연결합니다.

3. 배포 파이프라인

npm run build
vercel deploy --prod

# 사내망 배포 예: Nginx + Docker
docker build -t registry.example.com/rag-webapp:latest .
docker push registry.example.com/rag-webapp:latest
kubectl rollout restart deployment rag-webapp

FastAPI 게이트웨이와 동일한 도메인/인증 체계를 사용하여 쿠키 기반 세션을 공유합니다.

UX 최적화를 위한 체크리스트

대화 히스토리

  • 세션별 대화 기록을 IndexedDB 또는 Redis에 저장해 재방문 시 복원
  • 즐겨찾기/핀 기능으로 중요한 Q&A를 별도 보관
  • 지원 티켓, 사용자 피드백과 연동해 개선 포인트 수집

근거 문서 표시

  • 답변 하단에 파일명·라인값 등 출처 정보를 카드 형태로 제공
  • PDF/코드 뷰어를 내장해 즉시 열람 가능하게 구성
  • 하이라이트 문장에 hover 시 전체 문단 툴팁 표시

오류 처리 UX

  • LLM 타임아웃 시 재시도 버튼과 가이드 문구 제공
  • API 실패 시 이전 답변을 임시로 보여주고 Trace ID를 노출
  • 서버 오류 코드는 사용자 친화적 메시지로 변환

다국어/테마 지원

  • react-i18next로 한국어/영어 UI를 제공
  • 라이트/다크 테마를 시스템 설정과 연동
  • 모바일·태블릿·데스크톱 반응형 레이아웃 세분화

RAG 시스템에서의 장점

사용자 경험 향상

스트리밍과 근거 표시로 신뢰도 높은 답변을 빠르게 제공해 만족도를 높입니다.

업무 효율 상승

대화 히스토리와 즐겨찾기로 반복 조회를 줄이고, 협업 시에도 일관된 정보를 제공합니다.

운영 통합성

FastAPI 인증/로그를 재사용해 운영 레이어와 통합 관리가 가능합니다.

확장 유연성

플러그인 구조로 새로운 도메인(FAQ, 문서 검색)을 손쉽게 추가할 수 있습니다.
