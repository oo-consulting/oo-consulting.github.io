Parser 구축 가이드

Tree-Sitter 파서를 이용해 프로젝트의 소스코드를 추상 구문 트리(AST)로 분석하고, KorBERT · ChromaDB 단계에서 활용할 구조화 데이터를 준비합니다.

개발 순서

1단계 · Parser

언어별 AST 추출 · 품질 검증 자동화

단계 개요

Tree-Sitter는 다양한 언어의 소스코드를 AST(Abstract Syntax Tree)로 변환해 주는 경량 파서입니다. 단순 문자열 검색 대신 함수·클래스·주석 등 의미 단위로 코드를 이해할 수 있어 RAG 검색 정밀도를 크게 끌어올립니다.

Parser 단계에서 생성한 구조화 데이터는 KorBERT 임베딩 및 ChromaDB 적재의 입력이 되므로, 신뢰할 수 있는 토큰화 기준과 테스트 자동화를 갖춰 두는 것이 중요합니다.

구현 상세

구분 내용 비고
지원 언어 Java, JavaScript, Python, SQL, HTML/CSS 등 필요 시 tree-sitter-* 모듈 추가
출력 포맷 AST, S-expression, 토큰 리스트 임베딩 및 검색에 필요한 메타데이터 동시 생성
품질 검증 pytest + golden 파일 비교 리팩토링 시 AST 변화 감지용
배포 환경 Docker 기반 CI 구축 언어별 파서 빌드 캐시 활용

설치 및 기본 사용

1. 패키지 설치

pip install tree-sitter
pip install tree-sitter-java tree-sitter-python
pip install tree-sitter-sql tree-sitter-javascript

2. 예제 코드

from tree_sitter import Language, Parser
import tree_sitter_java as tsjava

JAVA_LANGUAGE = Language(tsjava.language())
parser = Parser(JAVA_LANGUAGE)

source = b"""
public class Sample {
    public void hello() {
        System.out.println("Hello World");
    }
}
"""

tree = parser.parse(source)
print(tree.root_node.sexp())

운영 팁

  • 언어별 파서 빌드를 CI 파이프라인에 포함시키고 테스트 유무를 확인합니다.
  • AST 변환 결과를 JSON/NDJSON 형태로 저장해 추후 재임베딩 시 재사용합니다.
  • 리팩토링 시 AST diff를 활용하면 코드 구조 변경을 빠르게 파악할 수 있습니다.
