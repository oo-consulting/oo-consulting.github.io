Parser

Tree-Sitter 파서를 활용해 소스코드를 추상 구문 트리(AST)로 정밀하게 분석하고, RAG 파이프라인이 활용할 구조화 데이터를 준비합니다.

개발 순서 1단계

개요

Tree-Sitter는 다양한 언어의 문법을 AST(Abstract Syntax Tree) 형태로 변환하여 구조적으로 이해할 수 있게 해주는 파서 프레임워크입니다. 단순 텍스트 검색을 넘어, 함수·클래스·주석 등 의미 단위로 코드를 분해할 수 있어 RAG 검색 정확도를 크게 높입니다.

본 단계에서는 공통 파서 환경을 구축하고, 이후 KorBERT 임베딩과 ChromaDB 구축에 활용할 인덱싱 단위를 정의합니다. 또한 테스트용 파이프라인을 마련해 파싱 품질을 빠르게 검증할 수 있도록 합니다.

구현 상세

항목 설명 비고
파서 엔진 Tree-Sitter (Python 바인딩) LLM 파이프라인과 동일한 Python 런타임에서 활용
지원 언어 Java, JavaScript, Python, SQL, HTML/CSS 등 필요 시 추가 그래머 모듈 설치
출력 포맷 AST, S-expression, 토큰/심볼 테이블 KorBERT 임베딩 전처리 입력으로 사용
검증 pytest + golden 파일 비교 리팩토링 시 AST 변화 감지

설치 및 기본 사용

1. 파이썬 의존성 설치

pip install tree-sitter
pip install tree-sitter-java tree-sitter-python
pip install tree-sitter-sql tree-sitter-javascript

2. 간단한 파싱 예제

from tree_sitter import Language, Parser
import tree_sitter_java as tsjava

JAVA_LANGUAGE = Language(tsjava.language())
parser = Parser(JAVA_LANGUAGE)

source = b"""
public class Sample {
    public void hello() {
        System.out.println("Hello World");
    }
}
"""

tree = parser.parse(source)
print(tree.root_node.sexp())

RAG 시스템에서의 이점

정확한 코드 이해

함수, 클래스, 주석을 파서가 자동으로 분리하여 의미 단위 검색이 가능합니다.

파이프라인 일관성

파서 출력 구조를 KorBERT, ChromaDB 단계와 공유해 재구현 비용을 줄입니다.

고품질 임베딩

선별된 AST 노드에 기반한 임베딩은 의미적 검색 정확도를 높입니다.

운영 편의성

테스트 자동화와 변경 감지로 레거시 코드도 안정적으로 관리할 수 있습니다.
