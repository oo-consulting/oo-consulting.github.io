Llama 3.3 & Qwen3 Coder 듀얼 통합

일반 질의는 Llama 3.3이, 코드 생성과 분석은 Qwen3 Coder가 담당하도록 라우팅해 품질과 효율을 모두 확보합니다. 프롬프트 체계, 컨텍스트 관리, Guardrail을 표준화합니다.

개발 순서

4단계 · LLM 통합

모델 라우팅 · 프롬프트 · Guardrail

단계 개요

RAG 파이프라인의 마지막 단계에서는 질의의 성격을 분류해 Llama 3.3과 Qwen3 Coder 중 적합한 모델을 선택합니다. 자연어 설명·정책 문서·요약은 Llama 3.3, 코드 생성·리팩토링·테스트 작성은 Qwen3 Coder가 처리하도록 분리합니다.

프롬프트 템플릿을 표준화하고, 컨텍스트 길이와 금지어 정책을 정의하며, Guardrail을 설정해 안전한 응답을 보장합니다.

구현 상세

항목 내용 비고
모델 라우팅 자연어/코드 질의 분류 후 Llama 또는 Qwen 선택 키워드·태그·confidence 기반 규칙
프롬프트 구조 System · User 템플릿 분리 톤·출력 포맷·금지어 정의
컨텍스트 관리 Top-K 슬라이싱 + 코드/주석 가중치 토큰 한도 대비 버퍼 확보
Guardrail 금지 패턴 필터링, 백업 모델 준비 재프롬프트/빈 응답 처리

설치 및 개발 방법

1. 모델 실행 환경

# Ollama 예시
curl https://ollama.ai/install.sh | sh
ollama pull llama3.3
ollama pull qwen3-coder

2. 라우터 서비스

python services/llm_router.py \
  --llama-endpoint http://localhost:11434/v1 \
  --qwen-endpoint http://localhost:11500/v1 \
  --rules configs/router_rules.yaml

3. 프롬프트 검증

python tests/prompt_validation.py \
  --template templates/system_prompt.md \
  --queries datasets/llm_eval.json \
  --llm llama

운영 체크리스트

컨텍스트 리라이트

  • 중복 문단 제거, 핵심 코드만 발췌
  • 질의 임베딩으로 관련 주석 매칭
  • 긴 컨텍스트는 슬라이딩 윈도우

Guardrail

  • 금지 키워드/패턴 필터
  • 코드 실행 예제는 샌드박스
  • 피드백 및 Trace ID 저장

토큰 관리

  • 시간대별 모델/파라미터 조정
  • 답변 길이 제한과 요약 설정
  • 토큰 비용 모니터링

장애 대응

  • 지연 시 백업 모델 호출
  • 빈 응답 발생 시 재프롬프트
  • 오류 메시지를 사용자 친화적으로 변환
