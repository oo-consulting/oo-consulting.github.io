Llama 3.3 & Qwen3 Coder 듀얼 스택

범용 질의에는 Llama 3.3, 코드 관련 요청에는 Qwen3 Coder를 활용하여 상황별 최적의 응답을 만드는 듀얼 LLM 전략을 구축합니다.

현재 진행률

65% · 라우팅 정책 검증 중

프롬프트 템플릿 정교화 & 토큰 비용 모니터링

개요

RAG 파이프라인의 마지막 핵심은 LLM 응답 품질을 안정적으로 유지하는 것입니다. 일반 질의·정책 문서는 Llama 3.3이, 코드 생성·리팩토링은 Qwen3 Coder가 담당하도록 역할을 분리하여 정확도와 속도를 동시에 확보합니다.

엔드포인트 라우팅, 프롬프트 표준화, 토큰 비용 관리, Guardrail 설계까지 포함한 운영 체계를 갖추어야 합니다.

구현 상세

항목 내용 비고
모델 라우팅 질의 분류 → Llama or Qwen3 선택 키워드, 태그, Confidence 기반 규칙
프롬프트 System / User 템플릿 분리 톤, 금지어, 코드 포맷 명시
컨텍스트 관리 토큰 창 대비 Top-K 슬라이싱 코드/문장 별 가중치 적용
Guardrail 금지 패턴·보안 키워드 필터 LLM 재시도/백업 모델 호출

설치 및 개발 방법

1. 모델 러너 준비

# Ollama 예시
curl https://ollama.ai/install.sh | sh
ollama pull llama3.3
ollama pull qwen3-coder

2. 질의 라우터 서비스

python services/llm_router.py \
  --llama-endpoint http://localhost:11434/v1 \
  --qwen-endpoint http://localhost:11500/v1 \
  --rules configs/router_rules.yaml

3. 프롬프트/출력 검증

python tests/prompt_validation.py \
  --template templates/system_prompt.md \
  --queries datasets/llm_eval.json \
  --llm llama

운영 체크리스트

컨텍스트 리라이트

  • 중복 문단 제거, 핵심 코드만 추출
  • 질의 → KorBERT 임베딩 → 근접 주석 결합
  • 긴 컨텍스트는 토큰 한도에 맞춰 압축

Guardrail

  • 금지 키워드/정책 위반 감지 시 재프롬프트
  • 코드 실행 예제는 read-only 샌드박스 이용
  • 사용자 피드백(👍/👎)과 Trace ID 기록

토큰 비용

  • 컨텍스트 길이 기반으로 Llama/Qwen 선택 가중치 조절
  • 답변 길이 제한을 시나리오별로 설정
  • 비업무 시간에는 경량 모델로 자동 전환

장애 대비

  • 모델 응답 지연 시 백업 모델 호출
  • 빈 응답 발생 시 ChromaDB 재쿼리 후 재시도
  • 오류 메시지는 사용자 친화적으로 변환
