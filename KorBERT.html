KorBERT 임베딩 파이프라인

Tree-Sitter에서 추출한 구조화 데이터를 KorBERT 입력 형식으로 정제하고, 대량의 한국어 코드·문서 임베딩을 자동으로 생성합니다.

개발 순서

2단계 · KorBERT

SentencePiece · GPU 배치 추론 · 품질 지표 수집

단계 개요

KorBERT는 한국어 문서와 코드 주석이 혼재된 데이터를 표현하기에 적합한 사전학습 언어 모델입니다. 파서가 제공한 AST 맥락을 문장 단위로 재조합하면, 코드 의미를 반영한 임베딩을 안정적으로 얻을 수 있습니다.

목표는 임베딩 생성·정규화·메타데이터 결합 과정을 자동화하고, 품질 지표(NDCG, 코사인 유사도)를 지속적으로 모니터링하는 것입니다.

구현 상세

구분 내용 비고
전처리 AST 기반 함수/주석 묶음을 문장 시퀀스로 변환 OOV 대비 사용자 사전 추가
토크나이저 SentencePiece · vocab 32K 코드 키워드 전용 토큰 강화
추론 환경 GPU (A100 기준 배치 32, 길이 256) FP16 저장으로 메모리 절감
후처리 L2 정규화 + 메타데이터 결합 ChromaDB 업로드용 JSONL 생성

설치 및 개발 방법

1. 의존성 설치

python -m venv .venv
source .venv/bin/activate  # Windows: .venv\Scripts\activate
pip install torch transformers sentencepiece accelerate
pip install kobert-transformers

2. 파이프라인 실행

python pipelines/korbert_embed.py \
  --input datasets/parser_units.jsonl \
  --model klue/bert-base \
  --batch-size 32 \
  --max-length 256 \
  --output artifacts/korbert_embeddings.jsonl

3. 품질 검증

python evaluations/embedding_metrics.py \
  --embeddings artifacts/korbert_embeddings.jsonl \
  --metric cosine --sample-size 2000

운영 체크리스트
