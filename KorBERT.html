개요

KorBERT 단계의 핵심 목표는 한국어 문서와 코드가 혼합된 데이터를 의미 벡터로 변환하여 검색과 질의 응답 정확도를 높이는 것입니다. Tree-Sitter가 제공한 AST 기반 구조 정보를 이용해 토큰 시퀀스를 재구성하고, 문맥 손실을 최소화한 상태에서 KorBERT 임베딩을 생성합니다.

파이프라인은 데이터 정규화, 토크나이징, 임베딩 추론, 후처리(정규화/라벨링) 네 단계로 구성되며, 벡터 품질 지표(NDCG, 코사인 유사도)를 자동 수집해 지속적인 개선이 가능하도록 설계되어 있습니다.

전처리 문장 정규화 & 토큰 증강 불필요한 공백/HTML 태그 제거, 사용자 사전으로 KorBERT 토크나이저에 코드 전용 토큰을 추가합니다.

결과 저장 메타데이터 & 임베딩 페어 파일 경로, 심볼 이름, 태그, 버전 정보를 함께 저장해 ChromaDB 연동 시 필터링과 버전 관리를 지원합니다.

설치 및 개발 방법

1. 환경 구성

# 가상환경 생성 및 활성화
python -m venv .venv
source .venv/bin/activate  # Windows: .venv\Scripts\activate

# 필수 패키지 설치
pip install torch transformers sentencepiece accelerate
pip install git+https://github.com/SKTBrain/KoBERT.git

KorBERT 모델은 Hugging Face Transformers와 SKT KorBERT 라이브러리를 함께 사용합니다. GPU 환경에서는 torch를 CUDA 버전에 맞춰 설치합니다.

2. 데이터 준비

python scripts/export_units.py \
  --tree-sitter-cache ./artifacts/ast \
  --include-comments \
  --output ./datasets/korbert_units.jsonl

python scripts/normalize_units.py \
  --input ./datasets/korbert_units.jsonl \
  --max-length 256 \
  --with-docstring

Parser 단계에서 생성된 AST 캐시를 기반으로 문맥 단위 샘플을 추출합니다. 코드와 주석, 문서가 한 묶음으로 구분되도록 전처리합니다.

3. 임베딩 파이프라인 실행

python pipelines/korbert_embedding.py \
  --model-name klue/bert-base \
  --input ./datasets/korbert_units.jsonl \
  --batch-size 16 \
  --output ./artifacts/embeddings \
  --use-half-precision \
  --metadata ./artifacts/metadata.json

추론 결과는 임베딩 벡터와 메타데이터 파일로 분리 저장합니다. 배치 크기와 half-precision 설정은 GPU 메모리 상황에 맞춰 조절 가능합니다.
