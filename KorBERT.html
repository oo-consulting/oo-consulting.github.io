KorBERT 임베딩 파이프라인

Parser 단계에서 추출한 코드/문서 토큰을 KorBERT 입력 형태로 정규화하고, 한국어 임베딩을 대량 생성하는 자동화 파이프라인을 구축합니다.

현재 진행률

85% · 품질 검증 중

배치 파이프라인 자동화 · NDCG 모니터링

개요

KorBERT는 한국어 문서와 코드 주석이 혼재된 데이터를 효율적으로 표현할 수 있는 사전학습 언어 모델입니다. 파서에서 추출된 AST 기반 토큰을 문장/함수 단위로 재조합해 임베딩을 생성하면, 코드 맥락까지 반영한 검색이 가능합니다.

본 단계의 핵심 목표는 임베딩 생성 자동화, 메타데이터 라벨링, 품질 지표 수집을 한 번의 파이프라인으로 묶는 것입니다. KorBERT 임베딩은 이후 ChromaDB에 적재되어 RAG 검색과 질의 응답의 토대가 됩니다.

구현 상세

항목 내용 비고
입력 전처리 AST 기반 함수/주석 묶음을 문장 시퀀스로 변환 OOV 토큰 대비 사용자 사전 등록
토크나이저 SentencePiece · 32K vocab 코드 키워드 전용 토큰 강화
추론 환경 GPU (V100/A100) · FP16 배치 추론 배치 크기 32, 시퀀스 256 토큰 기준
임베딩 후처리 L2 정규화 · 메타데이터 패키징 ChromaDB 업로드용 JSONL 생성

설치 및 개발 방법

1. 의존성 설치

python -m venv .venv
source .venv/bin/activate  # Windows: .venv\Scripts\activate
pip install torch transformers sentencepiece accelerate
pip install kobert-transformers

2. 파이프라인 실행

python pipelines/korbert_embed.py \
  --input datasets/parser_units.jsonl \
  --model klue/bert-base \
  --batch-size 32 \
  --max-length 256 \
  --output artifacts/korbert_embeddings.jsonl

3. 품질 검증

python evaluations/embedding_metrics.py \
  --embeddings artifacts/korbert_embeddings.jsonl \
  --metric cosine --sample-size 2000

운영 체크리스트

데이터 전처리

  • 함수/클래스 문맥 유지, 주석/Docstring 필수 포함
  • 민감 정보 제거 및 토큰화 전 정규화
  • 버전 태그, 파일 경로 등 메타데이터 동시 기록

품질 모니터링

  • 코사인 유사도 상위/하위 5% 수동 검토
  • NDCG, Mean Reciprocal Rank 주기적 집계
  • Feedback loop 기반 재학습 트리거 정의

성능/비용 최적화

  • GPU 배치 파라미터 자동 튜닝 스크립트
  • 임베딩 압축(quantization) 옵션 검토
  • 배치/스트리밍 파이프라인 구분 운영

거버넌스

  • 임베딩 버전과 소스 코드 버전을 동기화
  • PII/보안 정보가 포함된 샘플 자동 필터링
  • 모델/토크나이저 변경 시 영향도 보고
