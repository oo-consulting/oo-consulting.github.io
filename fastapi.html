FastAPI 기반 RAG API 게이트웨이

LlamaIndex QueryEngine을 REST/GraphQL/WebSocket API 형태로 노출하고, 인증·관측·배포 자동화를 갖춘 서비스 레이어를 제공합니다.

현재 진행률

55% · API 표준화 진행 중

JWT 인증, Rate Limit, OpenTelemetry 도입

개요

FastAPI 게이트웨이는 외부 애플리케이션이 RAG 기능을 안전하게 호출할 수 있도록 엔드포인트를 표준화합니다. REST와 WebSocket을 모두 지원하여 단문 질의와 스트리밍 응답을 커버하고, 인증/속도 제한/관측 기능을 통합합니다.

배포 환경은 Kubernetes를 기본으로 하며, CI/CD 파이프라인과 Observability 스택을 연동하여 운영 효율을 높입니다.

구현 상세

항목 내용 비고
API 엔드포인트 REST / GraphQL / WebSocket LLM 스트리밍은 WebSocket 사용
인증/권한 OAuth2 (JWT), API Key 역할 기반 접근 제어(Role Based)
관측 Prometheus · OpenTelemetry 지연, 토큰 사용량, 에러율 수집
배포 Kubernetes Rolling Update ConfigMap/Secret으로 설정 주입

설치 및 개발 방법

1. FastAPI 프로젝트 설정

python -m venv .venv
source .venv/bin/activate  # Windows: .venv\Scripts\activate
pip install fastapi uvicorn[standard] httpx pydantic-settings
pip install redis sqlalchemy prometheus-client

2. 라우터 예시

@router.post("/rag/query")
async def rag_query(request: QueryRequest, engine=Depends(get_query_engine)):
    response = await engine.aquery(request.to_prompt())
    return {"answer": response.response, "sources": response.source_nodes}

3. CI/CD 파이프라인

name: deploy-fastapi
on:
  push:
    branches: [ main ]
jobs:
  deploy:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v3
      - uses: actions/setup-python@v4
        with:
          python-version: '3.11'
      - run: pip install -r requirements.txt
      - run: pytest
      - uses: azure/k8s-deploy@v4
        with:
          manifests: manifests/deployment.yaml,manifests/service.yaml
          images: registry.example.com/rag-fastapi:${{ github.sha }}

운영 체크리스트

헬스 체크

  • /healthz · /readyz 엔드포인트 제공
  • LLM/ChromaDB 연결 실패 시 readiness fail
  • 지표 수집용 /metrics 노출

요청 제어

  • IP · API Key 기반 Rate Limit
  • 대형 요청은 비동기 큐(Kafka/Redis Streams)로 처리
  • 오류 응답에 Trace ID 포함

보안

  • HTTPS/TLS, mTLS 옵션 검토
  • 환경 변수는 Secret Manager 연동
  • API Key, 토큰은 주기적 회전

운영 지표

  • P95 응답 시간 1.5초 이하 유지
  • 429/5xx 비율 모니터링 및 알림
  • 토큰 사용량과 비용 리포트 자동 생성
