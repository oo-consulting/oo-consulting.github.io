FastAPI Gateway

RAG API 게이트웨이와 서비스 운영 자동화

개발 순서 6단계

개요

FastAPI 게이트웨이는 LlamaIndex 오케스트레이터를 API로 노출하고, 인증·속도 제한·관측 가능성(Observability)을 담당하는 중추 서비스입니다. REST/GraphQL 엔드포인트, WebSocket 스트리밍, 비동기 큐 연동까지 고려해 서비스 레이어를 구성합니다.

Kubernetes/KServe 환경 배포를 염두에 두고, 헬스 체크, 설정 주입, 롤링 업데이트 전략을 함께 설계하여 운영 부담을 최소화합니다.

API 게이트웨이 인증·속도 제한 관측 가능성 배포 자동화

구현 상세

구분 내용 상세 설명
API Endpoints REST + WebSocket 동기 요청은 REST로, 장시간 스트림 응답은 WebSocket을 제공해 UX를 개선합니다. LlamaIndex QueryEngine을 백엔드로 호출합니다.
인증·권한 JWT + API Key 사내 서비스는 JWT(OAuth2)로, 외부 파트너는 API Key로 접근하게 설계하고, 속도 제한(예: 60rpm)을 적용합니다.
관측 가능성 Prometheus + OpenTelemetry 요청당 latency, token usage, error rate를 수집하고, Trace ID를 LlamaIndex와 공유하여 전체 요청 흐름을 추적합니다.
배포 전략 K8s Rolling Update ConfigMap으로 모델/서비스 URL을 주입하고, 릴리스 전 Canary 트래픽으로 안정성을 점검합니다.

설치 및 개발 방법

1. FastAPI 프로젝트 구성

python -m venv .venv
source .venv/bin/activate  # Windows: .venv\Scripts\activate

pip install fastapi uvicorn[standard] httpx pydantic-settings
pip install redis sqlalchemy alembic

설정 관리를 위해 pydantic-settings를 사용하고, 세션 관리나 감사 로그 저장을 위해 데이터베이스 연동을 준비합니다.

2. LlamaIndex 연동 라우터

uvicorn app.main:app --reload

# app/routers/rag.py (예시)
@router.post("/rag/query")
async def rag_query(payload: QueryRequest, engine=Depends(get_query_engine)):
    response = await engine.aquery(payload.to_prompt())
    return {"answer": response.response, "sources": response.source_nodes}

비동기 호출을 지원하기 위해 aquery 등 Async API를 활용하고, 응답에 근거 자료(출처)를 포함합니다.

3. 배포 자동화 (예: GitHub Actions)

name: deploy-fastapi
on:
  push:
    branches: [ main ]

jobs:
  deploy:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v3
      - uses: actions/setup-python@v4
        with:
          python-version: '3.11'
      - run: pip install -r requirements.txt
      - run: pytest
      - uses: azure/k8s-deploy@v4
        with:
          manifests: manifests/deployment.yaml,manifests/service.yaml
          images: registry.example.com/rag-fastapi:${{ github.sha }}

CI 단계에서 테스트 통과 후 자동으로 컨테이너 이미지를 배포합니다. `manifests/` 폴더에서 리소스를 버전 관리합니다.

운영 자동화를 위한 체크리스트

헬스 체크

  • /healthz, /readyz 엔드포인트 구현
  • 쿼리 엔진 연결 실패 시 readiness fail로 설정
  • LLM/ChromaDB 의존성 점검 후 상태 코드 반환

요청 제어

  • IP 기반 Rate Limit, 요청 크기 제한(1MB)
  • 긴 쿼리는 비동기 큐(Kafka, Redis Streams)로 전환
  • 에러 발생 시 사용자 친화적 메시지와 Trace ID 제공

보안

  • HTTPS/TLS Termination, mTLS 옵션 검토
  • 모든 환경 변수는 Secret Manager에서 주입
  • 공격 패턴(IP, User-Agent)을 실시간 탐지

운영 메트릭

  • 응답 시간 SLA 준수 (예: P95 < 1.5s)
  • 토큰 소비량 추적 후 비용 보고서 발행
  • 주요 오류(429, 5xx) 비율이 임계값 초과 시 알림

RAG 시스템에서의 장점

안정적 API 제공

표준화된 엔드포인트를 통해 조직 내 다양한 애플리케이션이 RAG 기능을 활용할 수 있습니다.

운영 효율

모니터링과 자동화 체크리스트로 에러 대응 시간을 단축하고 예측 가능한 운영이 가능합니다.

보안 강화

인증/권한, 속도 제한, 비밀 관리가 통합되어 보안 사고 위험을 최소화합니다.

확장성

K8s와 연계한 롤링 업데이트 및 오토스케일링으로 트래픽 증가에 탄력적으로 대응합니다.
