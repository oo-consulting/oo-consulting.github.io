FastAPI 기반 RAG API 게이트웨이

LlamaIndex QueryEngine을 REST와 WebSocket API로 노출하고, OAuth2 인증, 속도 제한, 관측 기능을 통합해 안정적이고 확장 가능한 서비스 레이어를 구축합니다.

단계 개요

FastAPI 게이트웨이는 외부 애플리케이션이 RAG 기능을 안전하게 호출할 수 있도록 표준 API를 제공합니다. WebSocket 스트리밍을 통해 LLM 응답을 실시간으로 전달하고, OAuth2 기반 인증과 Rate Limit, Observability를 통합합니다.

Kubernetes 배포와 CI/CD 파이프라인을 연동하여 안정적인 운영과 빠른 업데이트를 지원합니다.

구현 상세

항목 내용 비고
엔드포인트 REST / GraphQL / WebSocket 제공 스트리밍 응답은 WebSocket
인증/권한 OAuth2 (JWT), API Key, ACL 역할 기반 접근 제어
관측 Prometheus · OpenTelemetry 지연, 토큰 사용량, 오류율 추적
배포 Kubernetes Rolling Update ConfigMap/Secret으로 설정 관리

설치 및 개발 방법

1. FastAPI 프로젝트 구성

python -m venv .venv
source .venv/bin/activate
pip install fastapi uvicorn[standard] httpx pydantic-settings
pip install redis sqlalchemy prometheus-client

2. 라우터 예시

@router.post("/rag/query")
async def rag_query(payload: QueryRequest, engine=Depends(get_query_engine)):
    response = await engine.aquery(payload.to_prompt())
    return {"answer": response.response, "sources": response.source_nodes}

3. CI/CD 파이프라인

name: deploy-fastapi
on:
  push:
    branches: [ main ]
jobs:
  deploy:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v3
      - uses: actions/setup-python@v4
        with:
          python-version: '3.11'
      - run: pip install -r requirements.txt
      - run: pytest
      - name: Deploy to Kubernetes
        run: kubectl rollout restart deployment rag-fastapi

운영 체크리스트

헬스 체크

  • /healthz · /readyz 엔드포인트 제공
  • LLM/ChromaDB 연결 실패 시 readiness fail
  • /metrics에서 Prometheus 지표 제공

요청 제어

  • IP · API Key 기반 Rate Limit
  • 대형 요청은 비동기 큐로 전환
  • Trace ID를 응답에 포함

보안

  • TLS/mTLS 적용, 비밀 값은 Secret Manager 사용
  • API Key·토큰 주기적 교체
  • 민감 정보 로그 마스킹

운영 지표

  • P95 응답 시간 1.5초 이하 유지
  • 429/5xx 비율 알림
  • 토큰 사용량·비용 리포트
